1) При навигации(кнопки вперёд/назад не в счёт) каждый раз создаётся новая ViewModel, по этой причине:
	- не сохраняется состояние страниц
	- в историю может быть добавлено две одинаковые страницы подряд, т.к проверка на равенство не срабатывает
	
	Возможное решение: Создавать вьюмодель с помощью статического метода с мемоизацией вместо конструктора, периодически чистить кэш объектов, по алгоритму LRE
	
2) При навигации по истории активный элемент в TreeView не обновляется.

3) При редактировании модели через несколько вьюмоделей создаётся бэкап всей модели, из-за этого, если в первой вьюмодели отменить редактирование, то результат редактирования второй будет потерян.

	Возможное решение: Делать бэкап только части модели, необходимой для вьюмодели. Его же и восстанавливать при отмене. (см. EditableViewBehaviour)
	
4) При редактировании переход на другую страницу без сохранения либо отмены должен выдавать предупреждение.
Сейчас результаты редактирования сохранятся, т.к редактируется непосредственно модель.
